4. Broadcast helper function (module-level, not exported):
   ```typescript
   function broadcast(wss: WebSocketServer, payload: object): void {
     const message = JSON.stringify(payload);
     for (const client of wss.clients) {
       if (client.readyState === WebSocket.OPEN) {
         client.send(message);
       }
     }
   }
   ```

5. Wire SessionManager events to WebSocket broadcast:

- Call `await sessionManager.disconnect(req.params.id)`
- Return: `res.json({ ok: true })`

Route 3: `POST /send`
- Accept JSON body: `{ repId: string, phone: string, message: string }`
- Validate all three fields are present and non-empty, return 400 if missing: `res.status(400).json({ error: 'repId, phone, and message are required' })`
- Get session: `const sock = sessionManager.getSession(repId)`
- If no session: `res.status(409).json({ error: 'Rep has no active session' })`
- Normalize phone: `const digits = phone.replace(/\D/g, '')` — validate length 7-15, return 400 if invalid
- Build JID: `const jid = jidEncode(digits, 's.whatsapp.net')` (import from @whiskeysockets/baileys)
- Send: `const result = await sock.sendMessage(jid, { text: message })`
- Return: `res.json({ ok: true, messageId: result?.key?.id ?? null })`

Add to the JavaScript section (runs after successful login):

```javascript
let ws = null;

function connectWebSocket() {
  if (ws) { ws.close(); }
  const protocol = location.protocol === 'https:' ? 'wss:' : 'ws:';
  ws = new WebSocket(protocol + '//' + location.host + '/ws?token=' + encodeURIComponent(token));

  ws.onmessage = (event) => {
    const msg = JSON.parse(event.data);
    if (msg.type === 'qr') {
      handleQrEvent(msg.repId, msg.dataUrl);
    }
    if (msg.type === 'status') {
      handleStatusEvent(msg.repId, msg.status);
    }
  };

  ws.onclose = () => {
    // Reconnect after 3 seconds if still logged in
    if (token) {
      setTimeout(connectWebSocket, 3000);
    }
  };

  ws.onerror = () => {
    // onclose will fire after onerror
  };
}
```

Call `connectWebSocket()` right after successful login (alongside `loadReps()`).

**QR Modal functionality:**

```javascript
let countdownInterval = null;
let activeQrRepId = null;

function handleQrEvent(repId, dataUrl) {
  // Only show QR for the rep we clicked Connect on
  if (repId !== activeQrRepId) return;

  const modal = document.getElementById('qr-modal');
  const img = document.getElementById('qr-img');
  const countdown = document.getElementById('qr-countdown');
  const statusText = document.getElementById('qr-status');

  img.src = dataUrl;
  statusText.textContent = 'Scan this QR code with WhatsApp on the rep\'s phone';
  modal.style.display = 'flex';

  // Reset 20-second countdown
  startCountdown(20);
}

function handleStatusEvent(repId, status) {
  // Update the rep's status badge in the list
  updateRepStatus(repId, status);

  // If this rep just connected and the QR modal is showing for them, close it
  if (repId === activeQrRepId && status === 'connected') {
    hideQrModal();
    // Show brief success feedback
    showToast('Rep connected successfully!');
  }
}

function startCountdown(seconds) {
  const countdown = document.getElementById('qr-countdown');
  let remaining = seconds;
  countdown.textContent = remaining + 's';

  if (countdownInterval) clearInterval(countdownInterval);
  countdownInterval = setInterval(() => {
    remaining--;
    countdown.textContent = remaining + 's';
    if (remaining <= 0) {
      clearInterval(countdownInterval);
      countdown.textContent = 'Expired — waiting for new QR...';
    }
  }, 1000);
}

function showQrModal(repId) {
  activeQrRepId = repId;
  const modal = document.getElementById('qr-modal');
  const img = document.getElementById('qr-img');
  const statusText = document.getElementById('qr-status');
  img.src = '';
  statusText.textContent = 'Requesting QR code...';
  modal.style.display = 'flex';
}

function hideQrModal() {
  activeQrRepId = null;
  const modal = document.getElementById('qr-modal');
  modal.style.display = 'none';
  if (countdownInterval) {
    clearInterval(countdownInterval);
    countdownInterval = null;
  }
}
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> WebSocket /ws | Token passed in query string; validated before upgrade |
| Browser -> POST /api/send | User-supplied phone number used to construct JID |
| Browser -> POST /api/reps/:id/connect | repId from URL used in sessionManager.connect() |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-06 | Elevation of Privilege | WebSocket /ws endpoint | mitigate | Token validated from query string in upgrade handler using timingSafeEqual; rejected connections get 401 + socket.destroy() |
| T-05-07 | Tampering | POST /api/send phone field | mitigate | Strip non-digits, validate length 7-15 characters, use jidEncode (established pattern from webhookHandler.ts) |
| T-05-08 | Information Disclosure | QR code broadcast | mitigate | QR broadcast only sent to authenticated WebSocket clients; unauthenticated connections rejected at upgrade |
| T-05-09 | Denial of Service | WebSocket reconnect loop | accept | 3-second reconnect delay prevents tight loop; internal tool with limited users |
| T-05-10 | Spoofing | WebSocket token in URL | accept | Token visible in server logs but not in browser address bar; acceptable for internal MVP (RESEARCH A3) |
</threat_model>
