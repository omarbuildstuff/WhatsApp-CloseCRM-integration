**requireBearer middleware:**
- Extract `Authorization` header, check it starts with `Bearer `
- Extract the token after `Bearer `
- Use `crypto.timingSafeEqual()` to compare the provided token against `config.dashboardPassword` (per RESEARCH security domain — timing-safe comparison)
- Handle length mismatch: if lengths differ, return 401 immediately (timingSafeEqual requires same-length buffers)
- On failure: respond `res.status(401).json({ error: 'Unauthorized' })` and return
- On success: call `next()`
- Import `timingSafeEqual` from `crypto` (already used in webhookHandler.ts)

**createDashboardRouter() function:**
Returns an `express.Router()` with these routes:

Route 1: `GET /` — Serve dashboard HTML
- Use `res.sendFile(path.join(__dirname, 'dashboard.html'))` to serve `src/dashboard.html`
- This route is NOT protected by requireBearer (public — the HTML has its own login screen)

Route 2: `GET /api/reps` — List all reps (protected)
- Query: `SELECT id, name, close_user_id, wa_phone, status, created_at FROM reps ORDER BY name`
- Return: `res.json(rows)`

Route 3: `POST /api/reps` — Add new rep (protected)
- Accept JSON body: `{ name: string, close_user_id?: string, wa_phone?: string }`
- Validate: `name` is required and non-empty (trimmed), respond 400 if missing
- Insert: `INSERT INTO reps (name, close_user_id, wa_phone) VALUES ($1, $2, $3) RETURNING *`
- Return: `res.status(201).json(row)`

Route 4: `DELETE /api/reps/:id` — Remove rep (protected)
- CRITICAL: Call `sessionManager.logout(repId)` BEFORE deleting the DB row (per RESEARCH Pitfall 4 — foreign key violation prevention)
- Delete: `DELETE FROM reps WHERE id = $1`
- Return: `res.json({ ok: true })`

Apply `requireBearer` to all `/api/*` routes using `apiRouter.use(requireBearer)` and mount apiRouter at `/api`. The `GET /` route must be outside the apiRouter.

Use `pino` logger (consistent with all other modules).

Wrap all route handlers in try/catch, returning `res.status(500).json({ error: 'Internal server error' })` on unexpected errors.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> API | Dashboard user sends Bearer token; must validate before any data access |
| Browser -> HTML | GET / serves HTML without auth; HTML itself contains no secrets |
| Webhook -> API | Close webhook at /webhook/close uses HMAC, not Bearer — must remain exempt from Bearer middleware |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Spoofing | requireBearer middleware | mitigate | Use crypto.timingSafeEqual for token comparison — prevents timing side-channel attacks |
| T-05-02 | Tampering | POST /api/reps | mitigate | Validate name is non-empty string; parameterized SQL queries ($1, $2, $3) prevent injection |
| T-05-03 | Elevation of Privilege | DELETE /api/reps/:id | mitigate | requireBearer middleware gates all /api/* routes; repId is parameterized in SQL |
| T-05-04 | Information Disclosure | GET / (dashboard HTML) | accept | HTML is public but contains no secrets; token stored in localStorage (acceptable for internal MVP per RESEARCH A3) |
| T-05-05 | Tampering | Rep deletion race | mitigate | sessionManager.logout(repId) called before DELETE FROM reps — prevents orphaned sessions with FK violations |
</threat_model>
