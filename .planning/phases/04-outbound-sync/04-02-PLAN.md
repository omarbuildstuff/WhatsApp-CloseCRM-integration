Modify `src/index.ts` to add the Close webhook route. This requires careful middleware ordering.

**Add import** at top of file:
```typescript
import { handleCloseWebhook } from './close/webhookHandler';
```

**CRITICAL MIDDLEWARE ORDERING:** The webhook route MUST be registered BEFORE `app.use(express.json())`. This is because `express.json()` consumes the raw request body and parses it into a JS object. The webhook handler needs the raw `Buffer` for HMAC signature verification.

**Add the webhook route** between `const app = express();` and `app.use(express.json());`:

```typescript
// Close webhook — MUST be before express.json() to preserve raw body for HMAC verification
app.post(
  '/webhook/close',
  express.raw({ type: 'application/json' }),
  (req, res) => {
    handleCloseWebhook(req, res).catch((err) => {
      logger.error({ err }, 'Webhook handler threw unexpectedly');
      if (!res.headersSent) res.status(500).end();
    });
  }
);
```

The resulting order in the `main()` function should be:
1. DB connectivity check
2. Session restoration
3. Message handler wiring
4. `const app = express();`
5. `app.post('/webhook/close', express.raw(...), handler)` -- NEW, before express.json()
6. `app.use(express.json());` -- existing, now AFTER webhook route
7. `app.get('/health', ...)` -- existing
8. `app.listen(...)` -- existing

The `express.raw({ type: 'application/json' })` is route-level middleware that only affects this specific route. It tells Express to keep `req.body` as a raw `Buffer` instead of parsing it as JSON. The handler function calls `handleCloseWebhook` and wraps it with a `.catch()` safety net for any unhandled promise rejection -- if the handler throws after sending 200, the error is logged but the client already got its response.
