**Create `src/whatsapp/sessionManager.ts`:**

**Imports:**
```typescript
import { EventEmitter } from 'events';
import makeWASocket, { DisconnectReason } from '@whiskeysockets/baileys';
import type { WASocket } from '@whiskeysockets/baileys';
import type { Boom } from '@hapi/boom';
import { usePgAuthState } from './authState';
import { pool } from '../db/pool';
import pino from 'pino';
```

**Note on @hapi/boom:** This is a transitive dependency of Baileys (already in node_modules). The `Boom` type is needed to extract `output.statusCode` from disconnect errors. Import as type-only (`import type`) so it does not need to be a direct dependency.

**Constants:**
- `const MAX_RECONNECT_ATTEMPTS = 10;`
- `const TERMINAL_REASONS = new Set([DisconnectReason.loggedOut, DisconnectReason.badSession, DisconnectReason.connectionReplaced, DisconnectReason.multideviceMismatch, DisconnectReason.forbidden]);`

**Class `SessionManager extends EventEmitter`:**

Private fields:
- `sessions = new Map<string, WASocket>()`
- `reconnectAttempts = new Map<string, number>()`
- `logger = pino({ level: 'info' })`

**`async connect(repId: string): Promise<void>`:**
1. Check for existing socket: `const existing = this.sessions.get(repId);` if exists, call `existing.end(undefined)` then `this.sessions.delete(repId)`
2. Call `const { state, saveCreds } = await usePgAuthState(repId, this.logger)`
3. Create socket: `const sock = makeWASocket({ auth: state, logger: this.logger.child({ repId }), printQRInTerminal: false })`
4. Register `sock.ev.on('connection.update', async (update) => { ... })`:
   - Destructure `const { connection, lastDisconnect, qr } = update`
   - If `qr`: emit `this.emit('qr', { repId, qr })`
   - If `connection === 'open'`: reset `this.reconnectAttempts.delete(repId)`, update DB `UPDATE reps SET status = 'connected' WHERE id = $1`, emit `this.emit('status', { repId, status: 'connected' })`
   - If `connection === 'close'`: call `await this.handleReconnect(repId, lastDisconnect)`
5. Register `sock.ev.on('creds.update', saveCreds)`
6. Register `sock.ev.on('messages.upsert', ({ messages, type }) => { if (type === 'notify') { for (const msg of messages) { this.emit('message', { repId, msg }); } } })`
7. Store: `this.sessions.set(repId, sock)`

**`private async handleReconnect(repId: string, lastDisconnect?: { error: Error | undefined }): Promise<void>`:**
1. Extract status code: `const statusCode = (lastDisconnect?.error as Boom)?.output?.statusCode;`
2. **Terminal branch** — if `statusCode !== undefined && TERMINAL_REASONS.has(statusCode)`:
   - Log: `this.logger.warn({ repId, statusCode }, 'Terminal disconnect — marking as needs_qr')`
   - Update DB: `UPDATE reps SET status = 'needs_qr' WHERE id = $1`
   - Call `await this.clearAuthState(repId)`
   - `this.sessions.delete(repId)`
   - Emit: `this.emit('status', { repId, status: 'needs_qr' })`
   - Return (do NOT reconnect)
3. **Reconnect branch** — else:
   - Get attempt count: `const attempt = this.reconnectAttempts.get(repId) ?? 0;`
   - If `attempt >= MAX_RECONNECT_ATTEMPTS`: log warning, update DB `UPDATE reps SET status = 'disconnected' WHERE id = $1`, `this.sessions.delete(repId)`, emit status 'disconnected', return
   - Calculate delay: `const delayMs = Math.min(2000 * Math.pow(2, attempt), 60_000);`
   - SPECIAL CASE: If `statusCode === DisconnectReason.restartRequired`, use `delayMs = Math.max(delayMs, 500)` (minimum 500ms to let saveCreds complete — per Pitfall 2)
   - Increment: `this.reconnectAttempts.set(repId, attempt + 1);`
   - Log: `this.logger.info({ repId, statusCode, attempt: attempt + 1, delayMs }, 'Scheduling reconnect')`
   - Schedule: `setTimeout(() => this.connect(repId), delayMs)`

**`async resumeAll(): Promise<void>`:**
- Query: `SELECT id FROM reps WHERE status IN ('connected', 'disconnected')` — explicitly EXCLUDE 'needs_qr' (per Pitfall 4)
- Log count of reps to resume
- For each row: `await this.connect(row.id)`

**`async disconnect(repId: string): Promise<void>`:**
- Get socket, if exists: call `sock.end(undefined)`, `this.sessions.delete(repId)`
- Update DB: `UPDATE reps SET status = 'disconnected' WHERE id = $1`
- Clear reconnect attempts: `this.reconnectAttempts.delete(repId)`

**`async logout(repId: string): Promise<void>`:**
- Get socket, if exists: call `await sock.logout()`, `this.sessions.delete(repId)`
- Call `await this.clearAuthState(repId)`
- Update DB: `UPDATE reps SET status = 'needs_qr' WHERE id = $1`
- Clear reconnect attempts: `this.reconnectAttempts.delete(repId)`

**`private async clearAuthState(repId: string): Promise<void>`:**
- `await pool.query('DELETE FROM wa_auth_keys WHERE rep_id = $1', [repId])`
- `await pool.query('DELETE FROM wa_auth_creds WHERE rep_id = $1', [repId])`

**`getSession(repId: string): WASocket | undefined`:**
- Return `this.sessions.get(repId)` (public getter for later phases to send messages)

**Export singleton:** `export const sessionManager = new SessionManager();`

```typescript
import express from 'express';
import { pool } from './db/pool';
import { sessionManager } from './whatsapp/sessionManager';
import { config } from './config';
import pino from 'pino';

const logger = pino({ level: 'info' });

async function main() {
  // Verify DB connectivity
  await pool.query('SELECT 1');
  logger.info('Database connected');

  // Restore all rep sessions from DB
  await sessionManager.resumeAll();
  logger.info('Sessions restored');

  // Minimal Express server (routes added in later phases)
  const app = express();
  app.use(express.json());

  app.get('/health', (_req, res) => {
    res.json({ status: 'ok' });
  });

  app.listen(config.port, () => {
    logger.info({ port: config.port }, 'Server started');
  });
}

main().catch((err) => {
  console.error('Startup failed:', err);
  process.exit(1);
});
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Application -> WhatsApp (Baileys) | Socket connections to WhatsApp servers carry auth credentials |
| Application -> PostgreSQL | Session state and credentials stored in remote DB |
| Timer -> Application | setTimeout callbacks trigger reconnection attempts |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Denial of Service | sessionManager.handleReconnect | mitigate | MAX_RECONNECT_ATTEMPTS=10 cap with exponential backoff (2s base, 60s max) prevents infinite reconnect loops; terminal reasons immediately stop reconnection |
| T-02-02 | Tampering | sessionManager SQL queries | mitigate | All queries use parameterized $1 placeholders — repId never interpolated into SQL |
| T-02-03 | Information Disclosure | pino logger | mitigate | Logger at info level; never log creds object or raw auth key values; Baileys child logger inherits info level |
| T-02-04 | Spoofing | sessionManager.connect | accept | repId comes from internal DB query (resumeAll) or future API call (Phase 5 will add Bearer auth); in Phase 1 there is no user-facing API to call connect() |
| T-02-05 | Elevation of Privilege | sessionManager.logout | accept | logout() permanently unlinks WhatsApp device; only callable from internal code in Phase 1; Phase 5 API will gate behind Bearer auth |
</threat_model>
