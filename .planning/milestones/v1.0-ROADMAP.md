## Overview

Self-hosted multi-rep WhatsApp integration for Close.com CRM replacing TimelinesAI ($25-50+/user/month). Each sales rep connects their WhatsApp via QR code scan, and all conversations sync to Close CRM as native WhatsApp Message activities. Bidirectional sync with mandatory loop guard prevents infinite message cycling.

## Milestone Summary

**Key Decisions:**
- Baileys @whiskeysockets/baileys@6.7.x (not v7 RC — confirmed 100% connection failure bug)
- PostgreSQL-backed auth state (never useMultiFileAuthState — container compatibility)
- Single HTML dashboard (no React, no build step)
- Bearer token auth (shared DASHBOARD_PASSWORD — no role separation for MVP)
- HMAC-SHA256 webhook verification with 5-minute replay guard
- Loop guard via external_whatsapp_message_id (non-negotiable, day one)

**Code Review Impact:**
- 26 findings fixed across 5 phases (8 critical, 18 warning)
- Notable fixes: TLS cert validation, reconnect timer cancellation, XSS prevention, concurrent request deduplication, idempotency guard

**Issues Deferred:**
- Human verification items: E2E outbound send, loop guard live test, dashboard visual check, QR modal live test
- dashboard.html path coupling (src/ vs dist/)
- Direction value inconsistency ('outgoing' vs 'outbound')
- Close API PUT vs PATCH for activity updates
